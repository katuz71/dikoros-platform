# 1. Корзина

Файл:

```text
app/(tabs)/cart.tsx
```

## 1.1 UI корзины

Корзина переделана в iHerb-style:

- компактный заголовок `Кошик (n)` по центру;
- карточка товара без лишних текстов, бренда, `Детальніше` и promo mismatch текста;
- фото, название, вариант/размер, количество, удалить, `Відкласти`, цена;
- quantity picker через bottom sheet;
- промокод сразу под товарами;
- секции `Відкладено` и `Мої списки` остаются внутри корзины;
- sticky checkout-кнопка внизу;
- пустое состояние показывает отложенные/списки, если они есть.

## 1.2 Отложенные и списки

`Відкласти`:

- не ведет на другую страницу;
- не добавляет в favorites;
- переносит товар в локальный `postponedItems`;
- удаляет товар из активной корзины;
- показывает товар на этой же странице во вкладке `Відкладено`.

`Мої списки`:

- показывает товары из `favorites` прямо в корзине;
- не открывает отдельную страницу;
- оранжевая кнопка корзины добавляет товар из списка в корзину.

## 1.3 Рейтинги

Фейковые звездочки удалены.

Рейтинг показывается только если у товара реально есть поля:

```ts
rating
averageRating
average_rating
totalReviews
total_reviews
review_count
reviews_count
```

Если данных нет — рейтинг не показывается вообще.

## 1.4 Рекомендации в корзине

Добавлен блок:

```text
Це може вас зацікавити
```

Визуально блок сделан в стиле iHerb: заголовок с линиями, сетка товаров, фото, цена, оранжевая кнопка корзины.

Логика рекомендаций персональная, не просто первые товары каталога:

- корзина — самый сильный сигнал;
- отложенные — второй сигнал;
- избранное / мои списки — третий сигнал;
- совпадение категории повышает товар;
- совпадение слов в названии повышает товар;
- похожий ценовой диапазон повышает товар;
- скидка дает небольшой бонус;
- реальные reviews/rating дают небольшой бонус;
- товары из корзины, отложенных и избранного не дублируются в рекомендациях.

# 2. Checkout

Файл:

```text
app/checkout.tsx
```

## 2.1 Основная логика

Checkout переделан в iHerb-style: основной экран не засорен длинной анкетой. Редактирование данных открывается в отдельных нижних окнах.

Основные блоки checkout:

- `Контакт`;
- `Доставка`;
- `Отримувач`;
- `Оплата`;
- `Ваше замовлення`;
- `Коментар`;
- `Підсумок`;
- sticky footer с суммой и кнопкой `Підтвердити`.

Каждый блок открывает отдельную модалку редактирования. Город/отделение НП/Укрпошты выбираются отдельным окном поиска.

## 2.2 Что визуально исправлено

- модалки стали выше;
- кнопки `Готово` подняты, чтобы их не перекрывал footer;
- окно города/отделения получило нижний safe-area запас;
- sticky кнопка заказа поднята над footer;
- у scroll checkout большой нижний отступ, чтобы summary и кнопка не перекрывались;
- в `Підсумок` добавлены товары: фото, название, вариант, количество, цена;
- галочка `Зберегти дані для наступних замовлень` включена по умолчанию.

## 2.3 Телефонная маска

В checkout добавлена маска телефона:

```text
+380 XX XXX XX XX
```

Применяется к:

- телефону покупателя;
- телефону получателя, если получает другой человек.

Поведение:

- при фокусе поле подставляет `+380`;
- ввод ограничен нужным количеством цифр;
- сохраненный телефон показывается в маске;
- перед отправкой номер валидируется;
- в backend и OneBox уходит чистый номер через `canonicalizePhone`, например `0999232030` / `380999232030` в зависимости от backend-нормализации.

## 2.4 Текст do-not-call

В модалке контакта текст изменен на:

```text
Не звонити, тільки повідомлення
```

Флаг продолжает уходить в заказ как:

```ts
do_not_call
```

И в OneBox как:

```text
customorder_Neperezvanivat = 1
```

# 3. Сохранение checkout-данных в профиль клиента

## 3.1 Backend

Добавлен новый безопасный router:

```text
routers/checkout_profile.py
```

Зарегистрирован в:

```text
main.py
```

Новые endpoints:

```text
GET /api/user/checkout-profile/me
PUT /api/user/checkout-profile/me
```

Они работают только по JWT текущего пользователя через `get_current_user_phone`.

## 3.2 Новые поля users

В `services/db_schema.py` добавлены idempotent migrations для таблицы `users`:

```text
last_name
middle_name
city_ref
warehouse_ref
recipient_name
recipient_phone
is_different_recipient
do_not_call
delivery_method
payment_method
checkout_comment
```

`fix_db_schema()` вызывается на startup, поэтому после `git pull` + restart backend новые поля добавляются автоматически.

## 3.3 Что сохраняется

При успешном заказе и включенной галочке `Зберегти дані для наступних замовлень` checkout сохраняет:

- имя;
- фамилию;
- отчество;
- email;
- способ связи;
- город;
- `city_ref`;
- отделение;
- `warehouse_ref`;
- ПІБ получателя;
- телефон получателя;
- получает ли другой человек;
- `do_not_call`;
- способ доставки;
- способ оплаты;
- комментарий checkout.

Сохранение расширенного профиля не блокирует заказ. Если PUT профиля упадет, заказ и OneBox-flow не ломаются.

## 3.4 Что подтягивается

При открытии checkout для авторизованного пользователя приложение грузит:

1. базовый профиль:

```text
GET /api/user/me
```

2. расширенный checkout-профиль:

```text
GET /api/user/checkout-profile/me
```

Если расширенный профиль есть, он перезаполняет checkout-данные.

## 4.1 Что запрещено менять без необходимости

Не трогать без реальной ошибки:

## 4.2 Цепочка заказа

Текущая цепочка:

```text
Checkout -> POST /create_order -> наша БД -> background task -> OneBox /api/orders/add -> OneBox update/set
```

Checkout отправляет в `/create_order`:

```ts

middle_name
client_full_name
recipient_name
recipient_phone
do_not_call
user_phone
phone
email
contact_preference
city
cityRef / city_ref
warehouse

delivery_method
items
totalPrice
payment_method
comment / comments
bonus_used
bonus_balance
use_bonuses
promo_code
promo_discount_percent
promo_discount_amount
save_user_data
guest_checkout
```

Backend затем отправляет заказ в OneBox.

## 4.3 Проверенный тестовый заказ

После внедрения checkout-профиля был сделан тестовый заказ.

Результат по логам:

```text

```

```text

```

```text

```

OneBox сначала вернул стандартную ошибку дубля телефона:

```text

```

Retry сработал правильно:

```text
[OneBox] Retry Response: {"result":"ok","orderId":82806}
```

Второй update OneBox прошел успешно:

```text
[OneBox] Official recipient update response: {"status":1,"dataArray":[82806]}
```

Профиль checkout сохранился:

```text
PUT /api/user/checkout-profile/me HTTP/1.1 200 OK
GET /api/user/checkout-profile/me HTTP/1.1 200 OK
```

Итог теста:


# 8. Manual QA перед публикацией Internal testing

Минимальный чеклист:

1. Открыть приложение авторизованным пользователем.
2. Проверить корзину: количество, удалить, отложить, мои списки, рекомендации.
3. Добавить товар в корзину.
4. Перейти в checkout.
5. Открыть `Контакт`, проверить маску телефона `+380 XX XXX XX XX`.
6. Заполнить контактные данные.
7. Открыть `Доставка`, выбрать город и отделение.
8. Открыть `Отримувач`, проверить сценарий покупатель сам / другой получатель.
9. Открыть `Оплата`, выбрать доступный способ.
10. Проверить `Підсумок`: товар, количество, цена, итог.
11. Убедиться, что кнопка `Підтвердити` видна над footer.
12. Оформить тестовый заказ.
13. Проверить OneBox: заказ создан, товар/количество/сумма/доставка/оплата корректные.
14. Вернуться в checkout и проверить, что данные подтянулись из профиля.
15. Проверить профиль и историю заказов.
